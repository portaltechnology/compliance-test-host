Solutions Overview
This section offers a comprehensive view of the blueprint, encompassing its objectives, constituent elements, as well as considerations and determinations related to its design for .

Purpose of the Blueprint
The primary goal of the Protected Utility blueprint is provide businesses with secure, flexible, and mobile solutions. Focused on security, the blueprint employs integrated native Microsoft technologies, ensuring a superior end-user experience.

Getting Started This overview serves as an ideal starting point for all users. The blueprint consists of three main components, namely:

  • Platform design This includes all supporting components for Windows 10, iOS, and Microsoft 365, including Microsoft Endpoint Manager.
  • Client devices design Encompassing components for Windows 10 and iOS.
  • Microsoft 365 design Incorporating configurations for Exchange Online, SharePoint Online, OneDrive for Business, and Teams.

The as-built-as-configured (ABAC) documents provide specific technical configurations for , outlining the environment settings and values based on the blueprint. These documents may utilize placeholders such as 'domainname.com.au' and 'tenant-name' when appropriate. However, they do not offer justifications for individual settings. There are two sets of ABAC documents, catering to the differences between configuring cloud and hybrid solutions.

Blueprint Components
The blueprint presents comprehensive insights into the following components, along with their rationale and configuration settings, enabling businesses to implement them effectively:

Cloud identity
This involves configuring Azure Active Directory, encompassing Multi-Factor Authentication (MFA) and conditional access, thereby facilitating secure logins from anywhere and the enforcement of appropriate security policies.

Microsoft 365
This entails the configuration of Exchange Online, SharePoint Online, Microsoft Teams, Whiteboard, Forms, and OneDrive for Business, facilitating cloud-based file storage.
